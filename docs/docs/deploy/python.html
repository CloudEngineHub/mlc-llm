Python API and Gradio Frontend¶

Table of Contents

  • Python API

    • Verify Installation

  • API Reference

  • Gradio Frontend

We expose Python API for the MLC-Chat for easy integration into other Python projects, we also provide a web demo based on gradio as an example of using Python API to interact with MLC-Chat.

Python API¶

The Python API is a part of the MLC-Chat package, which we have prepared pre-built pip wheels and you can install it by following the instructions in https://mlc.ai/package/.

Verify Installation¶

python -c "from mlc_chat import ChatModule; print(ChatModule)"

You are expected to see the information about the ChatModule class.

If the prebuild is unavailable on your platform, or you would like to build a runtime that supports other GPU runtime than the prebuilt version. Please refer our Build MLC-Chat Package From Source tutorial.

API Reference¶

User can initiate a chat module by creating ChatModule class, which is a wrapper of the MLC-Chat model. The ChatModule class provides the following methods:

class mlc_chat.ChatModule(target: str = 'cuda', device_id: int = 0)¶

Bases: object

__init__(target: str = 'cuda', device_id: int = 0)¶

Initialize a chat module.

Parameters:

  • target (str) – The target device type.

  • device_id (int) – The device id.

decode()¶

Decode the next token, the decoding result is stored in a buffer and can be retrieved by get_message().

embed(input: str, place_in_prompt: PlaceInPrompt = PlaceInPrompt.All)¶

Given a text input, get the embedding of the tokenized prompt. User can decide where to place the input in the prompt.

Parameters:

  • input (str) – The user input string.

  • place_in_prompt (PlaceInPrompt) – The place of the input message in the prompt.

get_message() → str¶

Get the output message in the current round.

Returns:

message

Return type:

str

Note

This function returns the message that corresponds to all the tokens decoded so far.

prefill(input: str, decode_next_token: bool = True, place_in_prompt: PlaceInPrompt = PlaceInPrompt.All)¶

Run prefill stage for a given input and optionally decode the first output token. User can decide where to place the input in the prompt.

Parameters:

  • input (str) – The user input string.

  • decode_next_token (bool) – Whether to decode the next token after prefilling.

  • place_in_prompt (PlaceInPrompt) – The place of the input message in the prompt.

prefill_with_embed(embedding: NDArray, decode_next_token: bool = True)¶

Given an embedding, run the prefill stage and optionally decode the first output token.

Parameters:

  • embedding (tvm.runtime.NDArray) – The embedding of user input.

  • decode_next_token (bool) – Whether to decode the next token after prefilling.

process_system_prompts()¶

Pre-process by prefilling the system prompts, running prior to any user input.

reload(lib: str, model_path: str, app_config_json: str = '')¶

Reload the chat module from the given library and model path.

Parameters:

  • lib (str) – The library path.

  • model_path (str) – The model path.

  • app_config_json (str) – The partial config that is used to partially override the model configuration.

reset_chat()¶

Reset the chat session and clear all chat history.

Note

The model remains the same after reset_chat(). To reload module, please use reload() instead.

reset_runtime_stats()¶

Reset the runtime stats.

runtime_stats_text() → str¶

Get the runtime stats text (encoding speed and decoding speed).

Returns:

stats – The runtime stats text.

Return type:

str

stopped() → bool¶

Check if the stop condition is met for the current round.

Returns:

stopped

Return type:

bool

Gradio Frontend¶

The gradio frontend provides a web interface for the MLC-Chat model, which allows user to interact with the model in a more user-friendly way. To use gradio frontend, you need to install gradio first:

pip install gradio

Then you can run the following code to start the interface:

python -m mlc_chat.gradio --artifact-path ARTIFACT_PATH --device-name DEVICE_NAME --device-id DEVICE_ID [--port PORT_NUMBER] [--share]
--artifact-path

The path to the artifact folder where models are stored. The default value is dist.

--device-name

The device name to run the model. Available options are: metal, cuda, vulkan, cpu. The default value is cuda.

--device-id

The device id to run the model. The default value is 0.

--port

The port number to run gradio. The default value is 7860.

--share

Whether to create a publicly shareable link for the interface.
